# Beginner HOWTO — QA Release Command Center Kit v2.1 FULL

1. Extract the ZIP.
2. Open `02_COMMAND_CENTER/index.html` in Chrome or Edge.
3. Create a workspace name and click Save Workspace.
4. Add one project.
5. Add one release.
6. Add risks, bugs, and sign-offs.
7. Export JSON regularly as a backup.
8. Use `04_TEMPLATES/` for editable QA templates.
9. Use `05_AI_PROMPTS/` for release analysis prompts.
10. Use `06_MARKETING/` for product page copy, social posts, and marketing images.

Important: this version stores data locally in the browser. Always export JSON backups.
